# Веб-сервис «Заявки в ремонтную службу»

Современное веб-приложение для управления заявками, разработанное с использованием **FastAPI (Python)**, **React (Vite + TailwindCSS v4)** и **PostgreSQL**.

## Как запустить проект

Проект полностью контейнеризован. Для запуска вам понадобится только установленный Docker.

1. Склонируйте репозиторий и перейдите в папку проекта.
2. Выполните команду:
   ```bash
   docker compose up -d --build
3. Приложение будет доступно по адресам:
   - Frontend (Пользовательский интерфейс): http://localhost:3000
   - Backend API (Swagger): http://localhost:8000/docs

Примечание: База данных инициализируется и заполняется тестовыми данными (сидами) автоматически при старте.

### Использование (Роли)

Для упрощения тестирования, система авторизации встроена в навбар на Frontend.
Просто выберите нужную роль в правом верхнем углу интерфейса:

      Клиент: Может создавать новые заявки.
      Диспетчер: Видит все заявки, фильтрует по статусам, назначает мастеров (ID: 1 или 2) или отменяет заявку.
      Мастер (ID: 1): Видит заявки, назначенные только на него. Может брать их в работу и завершать.

### Тестирование "Гонки" (Race Condition)

Защита от параллельного взятия заявки реализована через атомарный SQL-запрос UPDATE ... WHERE status = 'assigned'.

Для проверки выполните bash-скрипт (при запущенном Docker):

`bash race_test.sh`

Скрипт отправляет два одновременных curl запроса к API. Вы увидите, что один запрос будет успешен, а второй получит ошибку 409 Conflict.

## Автотесты

Для запуска автотестов бэкенда (проверка создания заявки и программная эмуляция гонки):
`docker compose exec backend pytest -v`
