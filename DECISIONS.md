
# Архитектурные решения (DECISIONS)

1. **Стек технологий:** Выбрана современная связка Python (FastAPI + SQLAlchemy) и React (Vite + Tailwind 4). Это обеспечивает высокую производительность, строгую типизацию (Pydantic/MyPy) и разделение слоев.
2. **База данных:** Использована PostgreSQL (через асинхронный драйвер `asyncpg`). Postgres отлично справляется с параллельными транзакциями.
3. **Решение проблемы "Гонки" (Race Condition):** Вместо использования тяжелых пессимистичных блокировок (`SELECT FOR UPDATE`), применен оптимистичный подход — **Атомарный UPDATE**. Статус обновляется только если текущий статус равен `assigned`. Если `rowcount == 0`, возвращается `409 Conflict`. Это самое быстрое и элегантное решение для подобных задач.
4. **Авторизация:** Упрощена согласно ТЗ. Чтобы не тратить время на настройку JWT и сосредоточиться на бизнес-логике, выбор пользователя реализован в UI через селектор состояния в React.
5. **Инфраструктура (Docker):** Приложение разбито на 3 независимых контейнера (`db`, `backend`, `frontend`). Фронтенд собирается в статику и раздается через легковесный `nginx`, что соответствует стандартам production-развертывания.
6. **Дизайн:** Реализована современная темная тема (Dark Mode) с использованием утилит Tailwind v4 для демонстрации навыков работы с актуальным UI/UX.
